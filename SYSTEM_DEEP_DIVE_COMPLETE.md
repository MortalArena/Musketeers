## نظام الجلسات (Session System)

### SessionContainer - الحاوية الكاملة للجلسة

**الموقع:** `pkg/session/container.go`

**المكونات الداخلية الـ 14:**

1. **Memory - الذاكرة الجماعية (4 أنواع)**
   - Episodic Memory: تخزن الأحداث (ماذا حدث؟ من فعل؟ النتيجة؟ الدروس المستفادة؟ الثقة؟)
   - Semantic Memory: تخزن الحقائق (ماذا نعرف؟ الفئة؟ الثقة؟ من صادق؟)
   - Procedural Memory: تخزن الطرق (كيف نفعل؟ الخطوات؟ معدل النجاح؟ متوسط المدة؟)
   - Meta Memory: تخزن الاستراتيجيات (كيف نفكر؟ متى نستخدم؟ كيف نستخدم؟ الفعالية؟)

2. **Skills - مدير المهارات**
   - يدير مستوى الوكلاء (0-100)
   - يدير شارات الإتقان (expert_{skill}, master, world_class)
   - يدير التطور (Evolution Count)
   - يدير المهارات الخاصة (Specializations)
   - يدير المهارات الفرعية (SubSkills)

3. **Workflow - محرك سير العمل**
   - يدير 16 مرحلة من سير العمل
   - يدير المهام في كل مرحلة
   - يدير التقدم (0-100)
   - يدير الاعتمادات بين المهام

4. **Artifacts - مخزن القطع الأثرية**
   - يخزن الملفات
   - يخزن البيانات
   - يخزن الروابط
   - يخزن البيانات الوصفية
   - يخزن Checksum (SHA256)

5. **Tasks - مدير المهام**
   - يدير قوائم الانتظار (pending)
   - يدير المهام الجارية (running)
   - يدير المهام المكتملة (completed)
   - يدير المهام الفاشلة (failed)
   - يدير الأولويات (Low, Medium, High, Urgent)
   - يدير الحالات (Pending, Assigned, Running, Completed, Failed, Cancelled)

6. **Progress - متتبع التقدم**
   - يدير نسبة الإنجاز (0-100)
   - يدير المخاطر (low, medium, high, critical)
   - يدير التأخيرات (الوقت المتوقع والفعلي)
   - يدير الإحصائيات (إجمالي المهام، المكتملة، المتأخرة، المعرضة للمخاطرة)

7. **Handoff - مدير التسليم**
   - يدير طلبات التسليم بين الوكلاء
   - يدير نقل القطع الأثرية
   - يدير التحقق من سلامة القطع الأثرية
   - يدير حالات التسليم (pending, accepted, rejected, completed, failed)

8. **Aggregator - مجمع الأجزاء**
   - يجمع المشروع النهائي
   - يدير هيكل الملفات (شجرة)
   - يدير الحجم الكلي
   - يدير وقت الإنشاء

9. **Reviewer - المراجع النهائي**
   - يفحص الجودة (هل اجتاز؟ النتيجة 0-100)
   - يدير المشاكل (critical, high, medium, low)
   - يدير الفئات
   - يدير المواقع
   - يدير الاقتراحات
   - يدير مدة المراجعة

10. **ChatManager - مدير المحادثة**
    - يدير الرسائل (thought, action, message, system)
    - يدير مصدر الرسالة (agent_did أو human)
    - يدير وقت الرسالة
    - يدير البيانات الإضافية (metadata)

11. **SessionBridge - جسر الجلسة**
    - يدير أنواع الجسور (one_way, two_way, multi, sync)
    - يدير حالة الجسر (idle, active, paused, error, closed)
    - يدير الرسائل المرسلة والمستلمة
    - يدير حجم البيانات المنقولة
    - يدير آخر نشاط

12. **SessionEventBus - ناقل أحداث الجلسة**
    - ينشر الأحداث
    - يستقبل الأحداث
    - يدير المشتركين
    - يدير قنوات التواصل

13. **UnifiedSessionState - الحالة الموحدة**
    - يوفر حالة موحدة للعميل
    - يحتوي على معلومات الجلسة
    - يحتوي على قائمة الوكلاء
    - يحتوي على قائمة المهام
    - يحتوي على معلومات التقدم
    - يحتوي على وقت التحديث

14. **BadgerDB - التخزين المستمر**
    - يخزن حالة الجلسة
    - يخزن الذاكرة
    - يخزن المهارات
    - يخزن القطع الأثرية
    - يخزن المحادثة

- الوكيل المدير (ManagerAgentID) - من AgentRegistry

- أنواع الوكلاء - coder, designer, tester, security

#### تكوين الوكلاء:
- مفتاح API لكل وكيل (مطلوب)
- النموذج لكل وكيل - claude-4.8, gpt-4, etc.
- القدرات المطلوبة - من AgentCapabilities

#### إعدادات الجلسة:
- تفضيلات العميل - من HumanClientPreferences
- معلومات الجهاز - من DeviceInfo
- معلومات الموقع - من LocationInfo

#### اختيار سيرفرات MCP:
- GitHub Server - لإنشاء issues و PRs
- Postgres Server - لتنفيذ SQL
- Slack Server - لإرسال رسائل
- تكوين مفاتيح API لكل سيرفر

### شاشة عرض الجلسة

#### معلومات الجلسة:
- الاسم والوصف
- الحالة (active, paused, completed, failed)
- نوع المشروع
- تاريخ الإنشاء والتحديث
- معرف الجلسة

#### الحالة الموحدة:
- قائمة الوكلاء (اسم، حالة، دور)
- قائمة المهام (عنوان، حالة، المسؤول، أولوية)
- معلومات التقدم (إجمالي المهام، المهام المكتملة، نسبة الإنجاز)
- وقت التحديث

#### المحادثة:
- الرسائل (thought, action, message, system)
- محتوى الرسالة
- مصدر الرسالة (agent_did أو human)
- وقت الرسالة
- البيانات الإضافية

#### المهام:
- قوائم الانتظار (pending)
- المهام الجارية (running)
- المهام المكتملة (completed)
- المهام الفاشلة (failed)
- معلومات المهمة (العنوان، الوصف، الأولوية، الحالة، المسؤول، المدخلات، المخرجات، الوقت المتوقع والفعلي)

#### سير العمل:
- المراحل (16 مرحلة)
- معلومات المرحلة (المعرف، الاسم، الوصف، الحالة، وقت البدء والإنجاز، المهام في المرحلة، نسبة التقدم)
- المهام في المرحلة (العنوان، الوصف، الحالة، المسؤول، الأولوية، التقدم، النتيجة، الاعتمادات)

#### التقدم:
- نسبة الإنجاز (0-100)
- المخاطر (low, medium, high, critical)
- التأخيرات (الوقت المتوقع والفعلي، مدة التأخير، السبب)
- إحصائيات (إجمالي المهام، المهام المكتملة، المهام المتأخرة، المهام المعرضة للمخاطرة)

#### الذاكرة:
- الأحداث (ماذا حدث؟ من فعل؟ النتيجة؟ الدروس المستفادة؟ الثقة؟)
- الحقائق (ماذا نعرف؟ الفئة؟ الثقة؟ من صادق؟)
- الطرق (كيف نفعل؟ الخطوات؟ معدل النجاح؟ متوسط المدة؟)
- الاستراتيجيات (كيف نفكر؟ متى نستخدم؟ كيف نستخدم؟ الفعالية؟)

#### المهارات:
- معلومات الوكيل (النوع، المستوى العام، المهارات الخاصة، شارات الإتقان، عدد التطورات)
- المهارات (الاسم والمستوى، الخبرة، معدل النجاح، المهارات الفرعية)
- شارات الإتقان (expert_{skill}, master, world_class)

#### القطع الأثرية:
- القطع الأثرية (الاسم، النوع، المسار، البيانات، Checksum، الحجم، من أنشأ؟، البيانات الوصفية)

#### حالة الوكلاء:
- قائمة الوكلاء المسجلين
- حالة كل وكيل (idle, busy, offline)
- الحمل (0-100)
- معدل النجاح
- عدد المهام الكلية والفاشلة
- القدرات

#### نشاط MCP:
- قائمة سيرفرات MCP
- حالة كل سيرفر (enabled, disabled)
- آخر استخدام
- سجل نشاط MCP
